なぜ「自分専用の中継器」が要るのか

Strava API を使うには Client Secret (秘密鍵) が必要です。 これを Web サービスの運営者サーバーに置く構成だと、運営者があなたの活動データに アクセスできてしまいます。

このツールには運営サーバーがありません。代わりに、あなた自身の Cloudflare アカウントに小さな中継プログラム (Worker) を設置していただきます。鍵はその Worker 内にのみ存在し、データは Strava ↔ あなたの Worker ↔ あなたのブラウザ の間だけで完結します。 開発者がデータに触れる経路は存在しません。

API キーを他人に預けるのは抵抗があるが、詳細な分析はしたい人のための構成です。 セットアップに手間が要るのは、それと引き換えです。

Strava の Fitness & Freshness を年度別に

年度ごとに切り替えて見る、
Fitness / Fatigue / Form チャート

Strava 公式 UI は時間軸スライダーで全期間を一度に表示します。 このツールは「2025 年だけ」「2018 年だけ」のように年度を切り替えてチャートを眺める SPA です。

このツールを使うための条件

  • サーバーを介しません ── データはあなたのブラウザと Strava の間だけでやり取りされます。
  • 自分専用の門番 (Worker) を置く ── セキュリティ上、あなた自身で Cloudflare 上に通信の中継役を立てる必要があります。
  • 設定は一度だけ ── 初回は面倒です。ただし一度終われば、あなたの秘密鍵を誰にも預けずに済みます。

取得したアクティビティはブラウザの localStorage に保存。サーバー送信はありません。

Strava API レート制限について
制限: 100 req / 15 分 · 1,000 req / 日 (個人 app 単位)。 1 年取得は通常 1〜2 req (per_page=200、年間 200 件超で 2)。 詳細値取得は 1 req/件 (年 100〜300 件なら 100〜300 req、15 分制限のため 30〜45 分掛かる)。
プライバシーと ToS について
✓ データはあなたのブラウザに localStorage でキャッシュされ、サーバには送られません。
✓ 一度取った年度は次回から即表示 (キャッシュから)、再取得は「最新に更新」ボタンで。
✓ Strava ToS §5.1 + §2.10 範囲内: 本人 OAuth + 本人 UI 表示のみ、第三者開示なし。
Strava ToS § 5.1 + § 2.10 準拠 ── 取得データはブラウザ内のみ保持、クロスユーザー共有なし、再配布なし。
