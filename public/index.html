Strava 年度別 Fitness & Freshness

走行データの保存先は、あなたのブラウザの localStorage と あなた自身がデプロイした Cloudflare Worker (OAuth 中継用) のみ。 そのための初回設定が約 10 分必要です。

設定は最初の一度だけ。あとは年度ボタンを押すだけです。

本ツールは「あなた専用の中継 (Cloudflare Worker)」を経由して Strava と通信します。Strava の Client Secret も走行データも、開発者 (yuuji) の サーバーは通らず、すべてあなたの管理下 (ブラウザ + あなたの Cloudflare アカウント) で完結します。初回設定 約 10 分。

なぜこれが必要か (technical)

Strava OAuth は Client Secret で code → access_token を交換する仕様。 Client Secret はブラウザ JS に置けない (公開 = 漏洩) ため、token 交換だけを行う薄い Worker を間に 挟む。Worker はあなたの Cloudflare 上に立つので、Secret も走行データも開発者のサーバーを通らない。 Worker のソースは worker/index.js (約 90 行) で公開。

  3. 2
    STEP 2 · 中継地点を建てる
    Cloudflare Worker をデプロイ
    下のコマンドを全部コピーしてターミナルで実行 (最終行の YOUR-USERNAME は自分の GitHub アカウントに置換)。 最後に表示される xxxx.workers.dev の URL を控える。 
    npm install -g wrangler
wrangler login
git clone https://github.com/yuujikamura/strava-pmc-viewer.git
cd strava-pmc-viewer/worker
wrangler secret put STRAVA_CLIENT_ID
wrangler secret put STRAVA_CLIENT_SECRET
wrangler deploy --var ALLOWED_ORIGIN:"https://YOUR-USERNAME.github.io"

    ↑ デプロイされる Worker のコード全文 (~90 行) は GitHub で公開 。やっているのは Strava の /oauth/token 中継のみ、データ保存ロジックは入っていません。

Strava API レート制限について
制限: 100 req / 15 分 · 1,000 req / 日 (個人 app 単位)。 1 年取得は通常 1〜2 req (per_page=200、年間 200 件超で 2)。 パワーデータの全件取得は 1 req/件 (年 100〜300 件なら 100〜300 req、Read 制限 100/15min のため 30〜45 分掛かる)。
プライバシーと ToS について
✓ データはあなたのブラウザに localStorage でキャッシュされ、サーバには送られません。
✓ 一度取った年度は次回から即表示 (キャッシュから)、再取得は「最新に更新」ボタンで。
✓ Strava ToS §5.1 + §2.10 範囲内: 本人 OAuth + 本人 UI 表示のみ、第三者開示なし。
Strava ToS § 5.1 + § 2.10 準拠 ── 取得データはブラウザ内のみ保持、クロスユーザー共有なし、再配布なし。
