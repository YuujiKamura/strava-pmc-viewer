初回セットアップ ── 自分の Strava と Cloudflare で動かす (20分)

このページは共有サーバを持ちません。あなたの Strava アカウントと、 あなたの Cloudflare アカウントで動かします。下記 4 ステップ完了で、 年度ごとのフィットネスチャートが見えるようになります。

  1. Strava で API アプリを作る (約 5 分)
    strava.com/settings/api を開いて「Create & Manage Your App」。Authorization Callback Domain にはこのページの host (—) を入れる。作成後の画面で Client ID と Client Secret を控える。
  2. Cloudflare Worker をデプロイする (約 10 分)
    Cloudflare アカウントを作成 → ターミナルで: 
    npm install -g wrangler
wrangler login
git clone <このリポ>
cd strava-pmc-viewer/worker
wrangler secret put STRAVA_CLIENT_ID       # ← step1 の Client ID を貼る
wrangler secret put STRAVA_CLIENT_SECRET   # ← step1 の Client Secret を貼る
wrangler deploy
    デプロイ完了画面に出る URL (例: https://strava-pmc-relay.xxxx.workers.dev) を控える。
  3. 下のフォームに 2 つの値を入れて「保存」 (約 1 分)
  4. 「Strava と接続」を押す → 認可画面で許可 → 年度ボタンが現れる

自分の Strava データで Fitness / Freshness を可視化

ヘッダの 「Strava と接続」を押すと Strava の認証ページに飛びます。 許可すると戻ってきて、年度ボタンから好きな年を選択 → その年のアクティビティだけ取得してFitness (CTL) / Fatigue (ATL) / Form (TSB) を計算・描画します。

✓ データはあなたのブラウザに localStorage でキャッシュされ、サーバには送られません。
✓ 一度取った年度は次回から即表示 (キャッシュから)、再取得は「最新に更新」ボタンで。
✓ Strava ToS §5.1 + §2.10 範囲内: 本人 OAuth + 本人 UI 表示のみ、第三者開示なし。

⚠ Strava API レート制限 (visitor 1 人 1 アプリ単位): 100 req / 15 分 ・ 1,000 req / 日。
1 年取得 ≈ 1〜2 req (per_page=200、年 200 件超だと 2 req)。 過去 10 年取り直しても合計 10〜20 req で、制限の足元にも届かないので普段は安心。
ただし 「詳細値を取得」は 1 req/件 ── 年 150 件なら 150 req、 15 分制限のため 30 分弱凍結を経由する。普段は無くても十分 (時間 × sport 係数で推定)。
