このアプリは、あなたのデータを外部に一切送信しません。 そのための初回設定 (Cloudflare Workers の Proxy デプロイ、約 10 分) が必要です。

API 接続設定 (初回のみ)

開発者のサーバーにデータを送らない仕組み (ローカル完結型) にするため、 Strava との通信を中継する Cloudflare Worker (Proxy) を、あなた自身のアカウントに デプロイしていただきます。

なぜこれが必要か (technical)

Strava OAuth は Client Secret で code → access_token を交換する仕様。 Client Secret はブラウザ JS に置けない (公開 = 漏洩) ため、token 交換だけを行う薄い Worker を間に 挟む。Worker はあなたの Cloudflare 上に立つので、Secret も走行データも開発者のサーバーを通らない。 Worker のソースは worker/index.js (約 90 行) で公開。

  2. →
  3. 2
    STEP 2
    Cloudflare にデプロイ
    下のコマンドを全部コピーしてターミナルで実行。最後に表示される xxxx.workers.dev の URL を控える。

wrangler deploy

Strava API レート制限について
制限: 100 req / 15 分 · 1,000 req / 日 (個人 app 単位)。 1 年取得は通常 1〜2 req (per_page=200、年間 200 件超で 2)。 詳細値取得は 1 req/件 (年 100〜300 件なら 100〜300 req、15 分制限のため 30〜45 分掛かる)。
プライバシーと ToS について
✓ データはあなたのブラウザに localStorage でキャッシュされ、サーバには送られません。
✓ 一度取った年度は次回から即表示 (キャッシュから)、再取得は「最新に更新」ボタンで。
✓ Strava ToS §5.1 + §2.10 範囲内: 本人 OAuth + 本人 UI 表示のみ、第三者開示なし。
Strava ToS § 5.1 + § 2.10 準拠 ── 取得データはブラウザ内のみ保持、クロスユーザー共有なし、再配布なし。
