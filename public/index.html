安心のための 3 つの準備

あなたの記録は Strava という大きな倉庫 にあります。そこから記録を借りてくる 「あなた専用の通訳さん」 を、3 つの準備で用意します。 通訳さんがいると、走った記録は あなたと通訳さんの間だけ で完結し、開発者の手元には 一切流れません。

  1. 1
    鍵を借りる
    Strava で「貸し出し窓口」に申し込む
    strava.com/settings/api の「Create & Manage Your App」を押すと、申込画面が出ます。 Authorization Callback Domain には — を入れて、 発行された Client ID と Client Secret (鍵 2 本) を控えてください。
  2. →
  3. 2
    通訳を雇う
    あなた専用の通訳さん (Worker) を用意する
    下のコマンドを 1 行ずつ実行すると、Cloudflare 上にあなただけの通訳さんが立ちます。 最後に表示される xxxx.workers.dev の URL を控えてください。 
    npm install -g wrangler
wrangler login
git clone <このリポ>
cd strava-pmc-viewer/worker
wrangler secret put STRAVA_CLIENT_ID
wrangler secret put STRAVA_CLIENT_SECRET
wrangler deploy

    ↑ デプロイされる Worker のコード全文 (~90 行) は GitHub で公開 。やっているのは Strava の /oauth/token 中継のみ、データ保存ロジックは入っていません。

  4. →
  5. 3
    画面に取り付ける
    この画面に「鍵」と「通訳の住所」を教える
    下のフォームに、1 で控えた Client ID と 2 で控えた Worker URL を入れて「保存」。あとはヘッダの 「Strava と接続」を押すだけで、あなたの走った記録が年度別グラフになって出てきます。
既定では公開アクティビティのみ取得 (権限を最小限に)。チェックすると private rides も読み取れます。

Data Status: Local Only 取得したアクティビティはあなたのブラウザ内 (localStorage) にのみ保存されます。外部サーバーへの送信は一切行いません。

Strava API レート制限について
制限: 100 req / 15 分 · 1,000 req / 日 (個人 app 単位)。 1 年取得は通常 1〜2 req (per_page=200、年間 200 件超で 2)。 詳細値取得は 1 req/件 (年 100〜300 件なら 100〜300 req、15 分制限のため 30〜45 分掛かる)。
プライバシーと ToS について
✓ データはあなたのブラウザに localStorage でキャッシュされ、サーバには送られません。
✓ 一度取った年度は次回から即表示 (キャッシュから)、再取得は「最新に更新」ボタンで。
✓ Strava ToS §5.1 + §2.10 範囲内: 本人 OAuth + 本人 UI 表示のみ、第三者開示なし。
Strava ToS § 5.1 + § 2.10 準拠 ── 取得データはブラウザ内のみ保持、クロスユーザー共有なし、再配布なし。
