なぜ Worker を立てるのか (信頼チェーン)
Strava OAuth は Client Secret で code → access_token を交換する。Client Secret は 絶対にブラウザ JS に置けない (フォーク・改造・スクレイピングで誰でも見える = 漏洩確定)。 よって「あなただけが触れる小さな中継器 (Worker)」を立てて、そこに Secret を置く。ブラウザは Worker 経由でしか token を取れない。
Cloudflare 側で行うこと: secret 保持 + CORS で あなたの origin だけ を許可 (`ALLOWED_ORIGIN`)。ブラウザ側で行うこと: token を localStorage に保管 + 全 API 呼び出しでこの token を Bearer で渡す。 この repo の Worker は wildcard CORS (*) を使わない実装になってる。

Strava API レート制限について
制限: 100 req / 15 分 · 1,000 req / 日 (個人 app 単位)。 1 年取得は通常 1〜2 req (per_page=200、年間 200 件超で 2)。 詳細値取得は 1 req/件 (年 100〜300 件なら 100〜300 req、15 分制限のため 30〜45 分掛かる)。
プライバシーと ToS について
✓ データはあなたのブラウザに localStorage でキャッシュされ、サーバには送られません。
✓ 一度取った年度は次回から即表示 (キャッシュから)、再取得は「最新に更新」ボタンで。
✓ Strava ToS §5.1 + §2.10 範囲内: 本人 OAuth + 本人 UI 表示のみ、第三者開示なし。
Strava ToS § 5.1 + § 2.10 準拠 ── 取得データはブラウザ内のみ保持、クロスユーザー共有なし、再配布なし。
